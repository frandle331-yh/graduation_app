# KAJILOG(カジログ)
 ふたりの家事を見える化します🏠

## サービス概要

同居する2人の家事・タスクの実行状況を可視化するWebアプリです。
誰がどの家事をどれくらいやっているかを履歴として残します。
偏りを数値で確認でき、自然な役割分担の見直しを促します。

## このサービスへの思い・作りたい理由

現代では共働きが主流となり、カップルや夫婦の共同生活において、
家事の負担をお互いに協力していくことが不可欠な時代だと考えています。

同居生活やカップル間では家事分担に関する不満が溜まりやすい一方で、
「やっていない」「ちゃんとやっている」といった主観的な主張が原因となり、
冷静な話し合いができなくなる場面が多いという意見を耳にしました。

そこで、「誰が・いつ・何をやったか」という事実を記録・可視化することで、
感情に左右されず、自然に分担を見直せる仕組みを作りたいと考えました。

このサービスを作ろうと考えた背景には、
これから私自身が同居生活を始めるにあたり、
家事分担によるすれ違いが起こらない共同生活を実現したいという思いがあります。

## ユーザー層について

同居しているカップル・夫婦（2人世帯） を主な対象としています。

家事分担が最も問題になりやすい生活形態であること、
また、2人に限定することで機能や設計をシンプルに保ち、
完成度の高いサービスを提供できると考えたためです。

まずは2人利用に限定し、将来的には
3人以上での分担にも対応できる拡張を検討しています。

## サービスの利用イメージ

ユーザーは世帯（グループ）を作成し、家事タスクを登録します。
家事を実行した際は、ワンタップで完了記録を残すだけです。

アプリは実行履歴を集計し、以下の内容をダッシュボードで可視化します。

どちらがどれくらい家事を担当しているか

家事の負担が偏っていないか

これにより、
「もっと家事手伝ってよ」
「私ばっかり家事してる」
といった不満を減らすことを目指します。

## ユーザーの獲得について

主に以下の方法を想定しています。

SNS（Xなど）での体験ベースの発信

「家事分担で揉めないために作ったアプリ」など、共感を軸にした投稿

ポートフォリオとしての公開

まずは「同じ悩みを持つ人に刺さる」ことを重視します。

## 競合サービスとの比較・差別化ポイント

家事関連アプリの多くは “やるべき家事の管理” を目的としており、特定の期間内で家事をどれだけ実行したかを後から振り返る仕組みは弱いという特徴があります。

本サービスは、既存アプリの提供価値を整理しながら、以下の点で明確な差別化を行っています。

### 1. 既存アプリの分類と特徴（競合調査）

| 分類                | 代表アプリ    | 主目的                | 弱点                        |
| ----------------- | -------- | ------------------ | ------------------------- |
| **家事ポイント・感謝可視化** | CAJICO   | ポイント獲得、ごほうび、感謝スタンプ | 家事の実績データを継続的に分析する設計ではない   |
| **掃除スケジュール管理** | PikaPika | 掃除のタイミング管理、リマインド   | “掃除限定”。家事全体のログや貢献度比較ができない |
| **家族の予定・家計統合**   | ファミリーログ  | カレンダー、TODO、家計簿     | 多機能だが家事の実行量を可視化する機能が弱い    |
| **チェックリスト＋習慣管理** | Sweepy   | チェックリスト、継続日数       | 行動ログは残るが詳細な分析や比較ができない     |
| **家事の時間・労力可視化**  | ペアワーク    | 家事時間の記録＆時給換算       | 記録の手間が大きく継続しづらい。ワンタップ入力不可 |
| **夫婦向けToDo共有**   | Cross    | “やること”管理、予定共有      | 完了済家事の振り返り設計が弱い           |

この調査から、既存アプリは「タスク管理」「予定管理」「ポイント」に特化し、
特定の期間内での実行量の蓄積とその振り返りに特化したアプリには、優位性があると考えています。

### 2. 本サービスの強み・差別化ポイント（家事サービス比較）
① “やった家事を可視化する” という領域に特化

多くのアプリは「これをやってね」という管理を行うものが多い
↓
本サービスは “これだけやったよ” を蓄積していくログ型

② ワンタップで記録完了 → 継続しやすい

既存アプリの中には入力するまでの工程が煩雑なものもある → 可能な限り簡略化

本サービスは：

タップ一回で記録

家事テンプレートから即登録

③ 家事の実績をデータで深く振り返れる

既存アプリでは弱い「分析」に特化。

月間家事量のグラフ

曜日別傾向

カテゴリー別集計

家族間比較（誰がどれだけやったか）

ヒートマップ表示


④ パートナーとの関係改善に直結する設計

感情を重視したCAJICO・ペアワークなど「感謝」系は存在するが、
客観的データによる「分析」系は少ない。

「やってるのに伝わらない」をデータで可視化

家事の分担の不公平を減らせる

関係性改善に寄与する


## 機能候補
### MVPリリース時に作りたい機能

本サービスの核となる実績の可視化を成立させるため、
MVPでは以下の機能を優先で実装します。

- 1. ワンタップ家事記録

既存アプリの弱点である「入力の手間」を解消し、
家事記録を継続しやすくする。

家事テンプレートをタップするだけで記録

タイトル入力・時間設定不要

ストレスなく毎日の実績が貯まる

- 2. 月別・週別の家事量集計

「どれだけやったか」を振り返りやすくする分析 UI の基盤。

月間家事量の集計グラフ

週次・月次の切り替え機能

カテゴリー別内訳（掃除・料理など）

曜日別の傾向（可能なら）

- 3. パートナー比較（貢献度の見える化）

“やってるのに伝わらない” を解消する。

世帯内ユーザーごとの実行数比較

カテゴリー別の貢献度

比率をグラフで表示

不公平感の解消＝関係改善につながる

- 4. 基本の世帯・ログイン周り

ユーザー登録・ログイン（認証機能）

世帯（グループ）作成・招待参加

- 5. 家事タスクの管理

家事タスクの登録・編集

実行履歴の一覧表示

ダッシュボード（担当割合・偏りの可視化）

### 本リリースまでに作りたい機能

- 通知機能
  - 未実行の家事タスクをリマインドする機能

- タスクの交代機能
  - 家事の実行ごとに担当者が自動で入れ替わる設定

- 家事テンプレートの一括登録
  よくある家事をあらかじめセットとして用意し、
  選択するだけで複数の家事タスクを一括で登録できる機能です。

  **家事テンプレート例**

  **「基本の家事セット」** を選択すると、以下のタスクが自動で登録されます。

  - 掃除機かけ（週2・交代）
  - ゴミ出し（週1・交代）
  - 洗濯（週3・交代）
  - 食器洗い（毎日・未指定）

- 期間別の集計切り替え（週・月）
  家事の実行状況を、期間ごとに切り替えて確認できる機能です。
  短期間の偏りと、長期間でのバランスを分けて把握できます。

  **集計表示例**

  **今週の集計**
  - Aさん：8回（60%）
  - Bさん：5回（40%）

  **今月の集計**
  - Aさん：30回（48%）
  - Bさん：32回（52%）


## 使用する技術スタック

| カテゴリ    | 技術                       | 選定理由                                    | 変更可能性                     |
| ------- | ------------------------ | --------------------------------------- | ------------------------- |
| フロントエンド | Ruby on Rails + Hotwire  | SPA化せず高速開発が可能。画面遷移が中心のためHotwireで十分対応できる | 中（将来的にReact/Next.js分離を想定） |
| バックエンド  | Ruby on Rails 7 / Devise | 学習済みスタックで実装スピード重視。認証・セキュリティが安定している      | 低                         |
| データベース  | PostgreSQL（Neon）         | 実行履歴や集計処理に強く、環境構築が容易                    | 低                         |
| インフラ    | Render                   | CI/CDが簡単で学習コストが低く、個人開発向き                | 低                         |
| 非同期処理   | Active Job               | 通知処理や集計処理を非同期で実行するため                    | 中                         |
| テスト     | RSpec / FactoryBot       | モデルロジックと集計処理の品質担保                       | 低                         |

| 機能          | 技術                         | 実装イメージ                 |
| ----------- | -------------------------- | ---------------------- |
| 家事実行ログの集計   | ActiveRecord / SQL         | `done_at` を基準に週・月単位で集計 |
| 家事分担の可視化    | Chart.js                   | 担当割合・偏りをグラフ表示          |
| 交代担当の自動割り当て | Rails Service Object       | 直近ログから次担当を決定           |
| 通知機能        | Active Job + Action Mailer | 未実行家事のリマインド            |
| 招待コード参加     | SecureRandom               | URLまたはコードで世帯参加         |
