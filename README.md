# KAJILOG(カジログ)
 ふたりの家事を見える化します🏠

## サービス概要

同居する2人の家事・タスクの実行状況を可視化するWebアプリです。
誰がどの家事をどれくらいやっているかを履歴として残します。
偏りを数値で確認でき、自然な役割分担の見直しを促します。

## このサービスへの思い・作りたい理由

現代では共働きが主流となり、カップルや夫婦の共同生活において、
家事の負担をお互いに協力していくことが不可欠な時代だと考えています。

同居生活やカップル間では家事分担に関する不満が溜まりやすい一方で、
「やっていない」「ちゃんとやっている」といった主観的な主張が原因となり、
冷静な話し合いができなくなる場面が多いという意見を耳にしました。

そこで、「誰が・いつ・何をやったか」という事実を記録・可視化することで、
感情に左右されず、自然に分担を見直せる仕組みを作りたいと考えました。

このサービスを作ろうと考えた背景には、
これから私自身が同居生活を始めるにあたり、
家事分担によるすれ違いが起こらない共同生活を実現したいという思いがあります。

## ユーザー層について

同居しているカップル・夫婦（2人世帯） を主な対象としています。

家事分担が最も問題になりやすい生活形態であること、
また、2人に限定することで機能や設計をシンプルに保ち、
完成度の高いサービスを提供できると考えたためです。

まずは2人利用に限定し、将来的には
3人以上での分担にも対応できる拡張を検討しています。

## サービスの利用イメージ

ユーザーは世帯（グループ）を作成し、家事タスクを登録します。
家事を実行した際は、ワンタップで完了記録を残すだけです。

アプリは実行履歴を集計し、以下の内容をダッシュボードで可視化します。

どちらがどれくらい家事を担当しているか

家事の負担が偏っていないか

これにより、
「もっと家事手伝ってよ」
「私ばっかり家事してる」
といった不満を減らすことを目指します。

## ユーザーの獲得について

主に以下の方法を想定しています。

SNS（Xなど）での体験ベースの発信

「家事分担で揉めないために作ったアプリ」など、共感を軸にした投稿

ポートフォリオとしての公開

まずは「同じ悩みを持つ人に刺さる」ことを重視します。

## サービスの差別化ポイント・推しポイント
差別化ポイント

既存のToDoアプリや共有カレンダーは、
「やること」を管理するものが多く、
「どれだけやっているか」を振り返る設計にはなっていません。

本サービスでは、
タスク管理ではなく、実績の管理・共有 を主目的としています。

家事の実行履歴を蓄積・可視化することで、
感覚ではなく事実をもとに分担を見直せる点が特徴です。

## 機能候補
### MVPリリース時に作りたい機能

- ユーザー登録・ログイン

- 世帯（グループ）作成・招待

- 家事タスクの登録・編集

- 家事実行の完了記録

- 実行履歴の一覧表示

- ダッシュボード（担当割合・偏りの可視化）

### 本リリースまでに作りたい機能

- 通知機能
  - 未実行の家事タスクをリマインドする機能

- タスクの交代機能
  - 家事の実行ごとに担当者が自動で入れ替わる設定

- 家事テンプレートの一括登録
  よくある家事をあらかじめセットとして用意し、
  選択するだけで複数の家事タスクを一括で登録できる機能です。

  **家事テンプレート例**

  **「基本の家事セット」** を選択すると、以下のタスクが自動で登録されます。

  - 掃除機かけ（週2・交代）
  - ゴミ出し（週1・交代）
  - 洗濯（週3・交代）
  - 食器洗い（毎日・未指定）

- 期間別の集計切り替え（週・月）
  家事の実行状況を、期間ごとに切り替えて確認できる機能です。
  短期間の偏りと、長期間でのバランスを分けて把握できます。

  **集計表示例**

  **今週の集計**
  - Aさん：8回（60%）
  - Bさん：5回（40%）

  **今月の集計**
  - Aさん：30回（48%）
  - Bさん：32回（52%）


## 使用する技術スタック

| カテゴリ    | 技術                       | 選定理由                                    | 変更可能性                     |
| ------- | ------------------------ | --------------------------------------- | ------------------------- |
| フロントエンド | Ruby on Rails + Hotwire  | SPA化せず高速開発が可能。画面遷移が中心のためHotwireで十分対応できる | 中（将来的にReact/Next.js分離を想定） |
| バックエンド  | Ruby on Rails 7 / Devise | 学習済みスタックで実装スピード重視。認証・セキュリティが安定している      | 低                         |
| データベース  | PostgreSQL（Neon）         | 実行履歴や集計処理に強く、環境構築が容易                    | 低                         |
| インフラ    | Render                   | CI/CDが簡単で学習コストが低く、個人開発向き                | 低                         |
| 非同期処理   | Active Job               | 通知処理や集計処理を非同期で実行するため                    | 中                         |
| テスト     | RSpec / FactoryBot       | モデルロジックと集計処理の品質担保                       | 低                         |

| 機能          | 技術                         | 実装イメージ                 |
| ----------- | -------------------------- | ---------------------- |
| 家事実行ログの集計   | ActiveRecord / SQL         | `done_at` を基準に週・月単位で集計 |
| 家事分担の可視化    | Chart.js                   | 担当割合・偏りをグラフ表示          |
| 交代担当の自動割り当て | Rails Service Object       | 直近ログから次担当を決定           |
| 通知機能        | Active Job + Action Mailer | 未実行家事のリマインド            |
| 招待コード参加     | SecureRandom               | URLまたはコードで世帯参加         |
