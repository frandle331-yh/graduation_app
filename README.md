## サービス概要

複数のAIキャラクターが、指定されたお題やシチュエーションに沿って会話を行う様子を閲覧できるサービスです。
ユーザーはAIと対話するのではなく、AI同士の自然な会話を「盗み聞き」する体験を楽しめます。
新しいエンタメに興味のあるユーザー向けの「合法的に他人の会話を盗み聞きできる」を実現します。

## このサービスへの思い・作りたい理由

日常生活の中で、電車内やカフェなどで他人の会話が偶然耳に入り、
その内容や関係性を想像してしまうことに面白さを感じた経験がありました。
カップルの会話や友人同士の何気ないやり取りには、勝手に聞くのは悪いけどなんだか気になって耳に入ってしまう不思議な魅力を感じることがあります。

現在の対話型AIサービスは「人間とAIが会話する」形式が主流ですが、
その一方で 「AI同士が会話する様子を第三者として眺める」という体験は、
まだ十分にサービス化されていないと感じました。
この「盗み聞きの楽しさ」と「AIによる会話生成」を組み合わせることで、
新しいエンタメ体験を提供したいと考え、このアプリケーションを考案しました。

## ユーザー層について

・新しいもの好き・AIエンタメに興味のあるユーザー
・自分で操作せず、眺めて楽しめる受動的コンテンツを好む
・雑談配信のネタを探しているユーザー

これらの層はアプリケーションの特性と強くマッチすると考えます。

## サービスの利用イメージ

ユーザーは会話のお題やシチュエーション、AIキャラクターの設定を選択します。
その後、AI同士が自動で会話を進行し、ユーザーはそれをリアルタイムで閲覧します。

自分が会話に参加する必要がないため、
作業中のBGM代わりや、配信のリアクション素材として気軽に利用できます。
「会話を覗き見る」という体験を通じて、
従来の対話型AIとは異なるエンタメ価値を得られます。

## ユーザーの獲得について

X（旧Twitter）での会話ログの一部切り抜き投稿
配信者による配信利用等
AIキャラクターやシチュエーションの面白さを前面に出した短尺動画展開

## サービスの差別化ポイント・推しポイント

### 差別化ポイント
ユーザーとAIが会話しない
AI同士の会話を「第三者視点」で楽しむ

既存の対話型AIサービスは、ユーザーの入力が体験の中心ですが、
本サービスでは「観察すること」自体が体験価値となります。

### 推しポイント

盗み聞きという人間の本能的な興味を活かしたコンセプト
シチュエーション次第で無限にコンテンツが生成される拡張性
配信・エンタメ用途に強く、二次利用しやすい構造

## 機能候補

### MVPリリースまでに実装予定

会話シチュエーション設定機能
AIキャラクター（2体）の簡易設定
AI同士のターン制会話生成
会話ログのリアルタイム表示
会話の再生成機能

### 本リリースまでに実装予定

キャラクター保存・管理機能
複数キャラクター（3体以上）対応
会話履歴の保存・共有
人気シチュエーションのランキング表示

## 使用する技術スタック

### フレームワーク

Ruby on Rails（最新版を想定）
フロントエンドはRails標準 + JavaScript

### データベース

PostgreSQL

### デプロイ先

Render または Heroku

### 使用予定のライブラリ・技術

OpenAI API（AI会話生成）

Hotwire / Turbo（リアルタイム表示）

WebSocket（会話の逐次更新）

AI API連携・リアルタイム通信・会話生成ロジックを中心に実装する予定です。
