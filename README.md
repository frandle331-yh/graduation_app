## サービス概要

同居する2人の家事・タスクの実行状況を、可視化するWebアプリです。
誰がどの家事をどれくらいやっているかを履歴として残します。
偏りを数値で確認でき、自然な役割分担の見直しを促します。

## このサービスへの思い・作りたい理由

現代では共働きが主流になり、カップルや夫婦の共同生活では、家事の負担を
お互いに協力していくことが不可欠な時代だと考えています。

同居生活やカップル間では、家事分担に関する不満が溜まりやすい一方で、
「やっていない」「ちゃんとやっている」といったお互いの主観的な主張が原因で
冷静な話し合いができなくなる場面が多いという意見を耳にしました。

「誰が・いつ・何をやったか」という事実を記録・可視化することで、
自然に分担を見直せる仕組みを作りたいと考えました。

このサービスが必要だと思った経緯としては、これから私自身が同居生活を始めるに
あたってこういったことですれ違いが起こらないような共同生活を作っていきたいと考えたからです。

## ユーザー層について

**同居しているカップル・夫婦（2人世帯）**を主な対象としています。

家事分担が最も問題になりやすい生活形態であるため

まずは2人利用に限定し、将来的な拡張（3人以上での分担）も考えています。

## サービスの利用イメージ

ユーザーは世帯（グループ）を作成し、家事タスクを登録します。
家事を実行した際はワンタップで完了記録を残すだけです。

アプリは実行履歴を集計し、
・どちらがどれくらい家事を担当しているか
・負担が偏っていないか
をダッシュボードで可視化します。

これにより
「もっと家事手伝ってよ」「私ばっかり家事してる」
を無くします

## ユーザーの獲得について

主に以下の方法を想定しています。

SNS（Xなど）での体験ベースの発信

「家事分担で揉めないために作ったアプリ」など共感を軸にした投稿

ポートフォリオの公開

まずは「同じ悩みを持つ人に刺さる」ことを重視します。

## サービスの差別化ポイント・推しポイント

差別化ポイント

既存のToDoアプリや共有カレンダーは
「やること」を管理するものが多く、
「どれだけやっているか」を振り返る設計にはなっていません。

タスク管理ではなく実績の管理、共有が主目的になります。

## 機能候補
MVPリリース時に作りたい機能

ユーザー登録・ログイン

世帯（グループ）作成・招待

家事タスクの登録・編集

家事実行の完了記録

実行履歴の一覧表示

ダッシュボード（担当割合・偏りの可視化）

本リリースまでに作りたい機能

通知機能（未実行タスクのリマインド）

タスクの交代機能（一回毎に担当が入れ替わる設定）

家事テンプレートの一括登録
具体例）

「基本の家事セット」を選ぶと、以下が一気に登録される：

掃除機かけ（週2・交代）
ゴミ出し（週1・交代）
洗濯（週3・交代）
食器洗い（毎日・未指定）

期間別の集計切り替え（週・月）
具体例）
「今週」：Aさん：8回（60%）
　　　　　Bさん：5回（40%）

「今月」：Aさん：30回（48%）
　　　　　Bさん：32回（52%）

## 使用する技術スタック

| カテゴリ    | 技術                       | 選定理由                                    | 変更可能性                     |
| ------- | ------------------------ | --------------------------------------- | ------------------------- |
| フロントエンド | Ruby on Rails + Hotwire  | SPA化せず高速開発が可能。画面遷移が中心のためHotwireで十分対応できる | 中（将来的にReact/Next.js分離を想定） |
| バックエンド  | Ruby on Rails 7 / Devise | 学習済みスタックで実装スピード重視。認証・セキュリティが安定している      | 低                         |
| データベース  | PostgreSQL（Neon）         | 実行履歴や集計処理に強く、環境構築が容易                    | 低                         |
| インフラ    | Render                   | CI/CDが簡単で学習コストが低く、個人開発向き                | 低                         |
| 非同期処理   | Active Job               | 通知処理や集計処理を非同期で実行するため                    | 中                         |
| テスト     | RSpec / FactoryBot       | モデルロジックと集計処理の品質担保                       | 低                         |

| 機能          | 技術                         | 実装イメージ                 |
| ----------- | -------------------------- | ---------------------- |
| 家事実行ログの集計   | ActiveRecord / SQL         | `done_at` を基準に週・月単位で集計 |
| 家事分担の可視化    | Chart.js                   | 担当割合・偏りをグラフ表示          |
| 交代担当の自動割り当て | Rails Service Object       | 直近ログから次担当を決定           |
| 通知機能        | Active Job + Action Mailer | 未実行家事のリマインド            |
| 招待コード参加     | SecureRandom               | URLまたはコードで世帯参加         |
